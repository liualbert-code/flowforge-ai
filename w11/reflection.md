\# Week 11 Reflection



\## 一、第一次自己定義客戶

我覺得這次比想像中難。



因為以前比較像是在做東西，

但這次是第一次真的站在「老闆」的位置思考：



「我到底想幫誰？」



最卡的地方是：



我開始發現，

不是所有人都適合服務。



你會開始思考：

\- 誰是公司真正想投入的人

\- 誰符合公司的 worldview

\- 誰適合這套 AI-native collaboration system



這跟單純做產品不一樣。



它比較像是在定義：

「這間公司的靈魂到底是什麼。」



我也開始理解，

真正的公司不只是提供功能，

而是選擇自己想服務的人與問題。



\## 二、公司遷移後還活著，代表什麼

我覺得這件事讓我開始理解：



真正重要的不是工具本身，

而是公司的 workflow、system 與 operating philosophy。



Gemini CLI 可以被替換，

但只要公司的：

\- collaboration structure

\- workflow

\- specialists

\- memory system

\- operating logic



還存在，



那這間公司其實就還活著。



這讓我開始感覺：



AI-native company 的本質，

比較像是一套「可遷移的工作系統」，



而不是綁定在某個模型、CLI 或工具上的東西。



因此真正重要的，

不是使用哪個 AI tool，



而是能不能建立：

\- 可持續運作的 workflow

\- AI-native collaboration structure

\- capability amplification system
